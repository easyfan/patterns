# patterns

Менеджер шаблонов рабочих процессов для Claude Code — список шаблонов, их инстанциирование в команды/агенты конкретного проекта и исправление отсутствующих шагов хуков через `--patch`.

```
/patterns                        # показать все доступные шаблоны
/patterns <pattern_name>         # инстанциировать шаблон рабочего процесса
/patterns --patch [command_name] # добавить отсутствующие шаги хуков в существующие команды
```

---

## Как работает

**Режим списка** (`/patterns`): читает `~/.claude/patterns/` и выводит каталог с описанием и датой последнего изменения. В конце отображается подсказка `💡` об опции `--patch`. В мета-проектах дополнительно показываются необработанные pending proposals.

**Режим инстанциирования** (`/patterns <name>`): загружает шаблон, автоматически определяет информацию о проекте (приоритет — `CLAUDE.md`, иначе — shell-зонды) и предзаполняет стартовый промпт. Затем создаёт соответствующие файлы `.claude/commands/` и `.claude/agents/`. Все сгенерированные файлы содержат поле `generated-from: <pattern_name>` в YAML front-matter для обеспечения трассируемости. Если целевой файл уже существует — предлагается выбор из трёх вариантов (перезаписать / пропустить / посмотреть и решить); без молчаливой перезаписи. По завершении предлагается опциональный контроль качества `/skill-review`, только если были записаны файлы команд или агентов.

**Режим патча** (`/patterns --patch`): сканирует существующие инстанциированные команды на наличие отсутствующих шагов хуков по точному совпадению заголовка раздела. Показывает план патча для подтверждения, затем добавляет недостающие шаги. Если патч не нужен — выводит положительное подтверждение.

## Установка

### Вариант A — Маркетплейс плагинов Claude Code (рекомендуется)

```
/plugin marketplace add easyfan/patterns
/plugin install patterns@patterns
```

> ⚠️ **Не покрыто автотестами**: `/plugin` — встроенная команда REPL Claude Code, не вызывается через `claude -p`. Запускать вручную в сессии Claude Code.

### Вариант B — Установочный скрипт

```bash
# macOS / Linux
git clone https://github.com/easyfan/patterns
cd patterns
./install.sh

# Windows
.\install.ps1
```

```bash
# Опции
./install.sh --dry-run      # предпросмотр изменений без записи
./install.sh --uninstall    # удалить установленные файлы
CLAUDE_DIR=/custom ./install.sh   # пользовательский путь к конфигурации Claude
```

> ✅ **Проверено**: покрыто pipeline тестирования навыков (looper Stage 5).

### Вариант C — Вручную

```bash
cp commands/patterns.md         ~/.claude/commands/
cp patterns/agent-monitoring.md ~/.claude/patterns/
```

> ✅ **Проверено**: покрыто pipeline тестирования навыков (looper Stage 5).

## Использование

```
/patterns [pattern_name | --patch [command_name]]
```

| Аргумент | Описание |
|----------|----------|
| _(нет)_ | Список всех шаблонов в `~/.claude/patterns/` |
| `<name>` | Инстанциировать указанный шаблон для текущего проекта |
| `--patch` | Сканировать все известные команды и добавить отсутствующие шаги хуков |
| `--patch <cmd>` | Применить патч только к указанной команде |

**Примеры:**

```
/patterns                              # каталог (с подсказкой --patch внизу)
/patterns agent-monitoring             # настроить мониторинг агентов в реальном времени
/patterns --patch research-module      # добавить недостающий хук контроля качества
/patterns --patch                      # пропатчить все известные команды
```

## Архитектура

```
/patterns:patterns (skill, координатор)
│
│  Разрешение путей (при каждом запуске):
│    PLUGIN_ROOT      = dirname(dirname($SKILL_FILE))
│    PLUGIN_TEMPLATES = $PLUGIN_ROOT/templates/   ← встроенные шаблоны в кеше plugin
│    USER_PATTERNS    = ~/.claude/patterns/        ← шаблоны, добавленные пользователем
│    (сканируются оба источника; пользовательские шаблоны имеют приоритет над встроенными)
│
├── Без арг.:  Bash — сканировать USER_PATTERNS + PLUGIN_TEMPLATES → список + подсказка --patch
│              (только мета-проект) Bash — сканировать pending proposals
│
├── <name>:    Читать шаблон (сначала USER_PATTERNS, затем PLUGIN_TEMPLATES)
│              Bash — определить информацию о проекте (CLAUDE.md или shell-зонд)
│              Заполнить плейсхолдеры стартового промпта
│              Шаг 5a — проверить существование файлов → 3 варианта: перезаписать/пропустить/посмотреть
│              Записать .claude/commands/<cmd>.md + агентов (при необходимости)
│              → Шаг 7: контроль качества /skill-review (только при новых файлах .claude/)
│
└── --patch:   Bash — найти установленные файлы команд
               Grep — обнаружить отсутствующие шаги хуков по точному заголовку
               Показать план патча → ожидать подтверждения
               Edit — добавить отсутствующие шаги хуков
               Сообщить о полном состоянии, если патч не нужен
```

## Практический пример: от шаблона к безопасности в реальном времени

24 марта 2026 года шаблон `agent-monitoring` был создан после того, как `/news-digest` показал статью OpenAI Engineering о мониторинге агентов в реальном времени. В следующей сессии в мета-проекте был запущен `/patterns agent-monitoring`:

1. **Шаблон загружен**: `agent-monitoring.md` прочитан из `~/.claude/patterns/`
2. **Проект определён**: найден `CLAUDE.md` → контекст проекта заполнен автоматически, без shell-зондов
3. **Созданы файлы**:
   - `~/.claude/commands/agent-monitoring-workflow.md` — команда-координатор для аудитов после задач
4. **Контроль качества**: пользователь запустил `/skill-review agent-monitoring-workflow` — 3 рекомендации, все применены в той же сессии

От инстанциирования шаблона до готовой к продакшену команды — менее 10 минут.

## Разработка

### Evals

`evals/evals.json` содержит 7 тест-кейсов для режимов списка, инстанциирования и `--patch`:

| ID | Промпт | Что проверяется |
|----|--------|----------------|
| 1 | `/patterns` | Список всех шаблонов с именами и датами изменения; подсказка `--patch` внизу |
| 2 | `/patterns` | Вывод подсказки по использованию; режим списка не вызывает `/skill-review` |
| 3 | `/patterns nonexistent_pattern_xyz_12345` | Сообщение «не найдено» и список доступных шаблонов; без сбоя |
| 4 | `/patterns --patch` | Сканирование команд с полем `generated-from`; результат или «не найдено» |
| 5 | `/patterns --patch research-module` | Результат сканирования для указанной команды |
| 6 | `/patterns` | Список шаблонов с подсказкой по использованию `--patch` внизу |
| 7 | `/patterns --patch nonexistent_cmd_xyz_99999` | Неизвестное имя команды → сообщение «не найдено»; без сбоя |
